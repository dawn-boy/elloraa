tell me what is fp8 and awq checkpoints mean?

---

## Assistant

Let's start from the ground up.

What a checkpoint is

A model is just a massive collection of numbers — billions of them — called weights. A checkpoint is those numbers saved to disk as files. When you "load a model," you're reading those numbers from disk into VRAM.

What quantization is

Every number in a model takes up space. By default, models are saved in BF16 — Brain Float 16 — which uses 16 bits per number. A 32B parameter model at BF16 = 32 billion × 2 bytes = ~64GB.

Quantization means storing those numbers in fewer bits. Fewer bits per number = smaller file = less VRAM needed. The trade-off is precision — you're rounding numbers to fit them into a smaller format, which can affect quality.

FP8

FP = Floating Point. 8 bits per number instead of 16.

Floating point means the format can represent a wide range of values — both very small decimals and large numbers — because it has an exponent component, like scientific notation. This matters because neural network weights span a wide range of magnitudes.

At FP8 you get:

FP8 is the closest you can get to BF16 quality while still halving the memory footprint.

AWQ

AWQ = Activation-Aware Weight Quantization. This is a specific algorithm for going down to 4 bits — INT4, which is integer format (only 16 possible values per number, no fractions).

INT4 is aggressive. Naively rounding every weight to 4 bits destroys quality. What AWQ does differently: before quantizing, it runs sample data through the model to figure out which weights matter most for output quality (based on how activations flow through the network). It then protects those critical weights from heavy rounding. The less important weights take the precision hit instead.

The result:

Side by side

FP8 is the sweet spot when your GPU supports it natively — you get half the memory cost with almost no quality loss and full hardware speed. AWQ is the choice when you need to go smaller and can accept the quality trade-off, or when the GPU doesn't have FP8 tensor cores.

- Half the size of BF16 (~32GB for a 32B model)
- Near-identical quality because floating point handles the value range well
- Native hardware acceleration on modern GPUs (H100, L40S) — the chip has dedicated FP8 tensor cores, so it doesn't have to convert anything at compute time

- ~4x smaller than BF16 (~16–18GB for a 32B model)
- Surprisingly good quality for 4-bit
- Slower at inference than FP8 — because the GPU has to dequantize INT4 back to a higher precision at compute time (modern GPUs don't have native INT4 matrix multiply hardware the way they do for FP8)
